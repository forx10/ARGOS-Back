# ARGOS Phase 1 Backend - Project Instructions

## Project Overview
ARGOS is a personal assistance system for managing rotating work shifts, automatic app/site blocking, and alarm scheduling. Phase 1 focuses on:
- Rotating shift analysis (6AM-2PM, 2PM-10PM, 10PM-6AM)
- Persistent app blocking (Instagram, TikTok, YouTube)
- Website blocking (pornography/adult sites)
- Automatic alarm scheduling (1.5h before shift)
- Tasker/AutoRemote webhook integration

## Technology Stack
- **Framework**: NestJS with TypeScript (strict mode)
- **Runtime**: Node.js 22.14.0
- **Database**: PostgreSQL with Prisma ORM
- **HTTP Client**: Axios (for Tasker webhook calls)
- **API Documentation**: Swagger/OpenAPI

## Database Schema
Three main tables:
- `turno`: Shift scheduling (id, usuario_id, hora_inicio, hora_fin, tipo_turno, fecha, activo)
- `bloqueo_activo`: Active blocks (id, usuario_id, apps_bloqueadas[], sitios_bloqueados[], tiempo_inicio, tiempo_fin, estado, razon)
- `sitio_bloqueado`: Blocked site categories (id, patron_url, categoria, activo)

## API Architecture
**4 Main Modules**:
1. **Turnos Module** - Shift management
   - POST /api/turnos - Create shift
   - GET /api/turnos/:usuarioId - Get user shifts
   - GET /api/turnos/:usuarioId/analizar - Analyze shifts and generate orders

2. **Bloqueo Module** - App/site blocking control
   - POST /api/bloqueo - Create block
   - DELETE /api/bloqueo/:bloqueoId - Complete/unblock
   - GET /api/bloqueo/:usuarioId/activos - Get active blocks

3. **Webhook Module** - Tasker/AutoRemote integration
   - POST /api/webhook/tasker - Listen for Tasker events
   - GET /api/webhook/tasker/estado/:bloqueoId - Get block status

4. **Estado Module** - Status queries
   - GET /api/estado/:usuarioId - Get complete status (blocks, shifts, alarms)
   - GET /api/estado/:usuarioId/bloqueos - Get active blocks only
   - GET /api/estado/:usuarioId/turnos - Get upcoming shifts (7 days)
   - GET /api/estado/sitios/bloqueados - Get blocked sites by category

## Key Business Logic
- **Alarm Calculation**: 1.5 hours (90 minutes) before shift start time
- **Automatic Blocking**: When user enters shift, auto-create block until shift end
- **Shift Detection**: Checks current time against registered shifts (same day)
- **Tasker Integration**: Sends JSON payloads to AutoRemote webhook with block/unblock commands
- **State Tracking**: All blocks have status: activo, pausado, completado

## Environment Variables
- `DATABASE_URL`: PostgreSQL connection (auto-configured)
- `USUARIO_ID`: Default user ID (default: usuario_1)
- `TASKER_WEBHOOK_URL`: AutoRemote webhook endpoint (optional - service logs warning if not set)

## Response Format
All endpoints return standardized JSON:
```json
{
  "success": true|false,
  "data": { /* endpoint-specific data */ },
  "message": "Human-readable status message"
}
```

## Tasker Webhook Payload Format
**Block Command**:
```json
{
  \"accion\": \"bloquear\",
  \"usuario_id\": \"usuario_1\",
  \"bloqueo_id\": 1,
  \"apps\": [\"Instagram\", \"TikTok\"],
  \"sitios\": [\"pattern.com\"],
  \"duracion_minutos\": 120,
  \"timestamp\": \"ISO-8601\"
}
```

**Tasker Response Event**:
```json
{
  \"usuario_id\": \"usuario_1\",
  \"bloqueo_id\": 1,
  \"evento\": \"bloqueo_confirmado|desbloqueo_confirmado|error_bloqueo\",
  \"timestamp\": \"ISO-8601\"
}
```
