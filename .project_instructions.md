# ARGOS - Sistema de Asistencia Personal

## Visión del Usuario (Actualizada - Abril 2026)

### Principios Clave:
- **ARGOS es activado por VOZ**: Todo se hace mediante comandos de voz, no acciones automáticas
- **Sistema inteligente de contexto**: ARGOS entiende comandos complejos en lenguaje natural
- **Recordatorios programados**: ARGOS puede crear alarmas y recordatorios para el futuro
- **Control granular**: El usuario decide cuándo activar cada función

---

## Funcionalidades Implementadas (Fase 1)

### 1. Sistema de Alarmas Inteligente
**Comandos de voz:**
- "ARGOS agregar una alarma a las 4:15 am para el turno 1"
- "ARGOS recuérdame a las 7pm que tengo que hacer ejercicio"
- "ARGOS ponme una alarma en 2 horas"

**Backend debe:**
- Parsear comandos de voz con LLM
- Crear alarmas programadas en DB
- Enviar notificación a Tasker cuando llegue la hora
- Tasker reproduce alarma + mensaje de voz

### 2. Contexto de Estudio (Bajo Demanda)
**NO automático**, solo cuando el usuario lo pide:

**Comandos de voz:**
- "ARGOS hoy a las 3 voy a estudiar, recuérdame que tengo que estudiar y bloquea toda distracción"
- "ARGOS mañana a las 9am tengo examen, bloquea redes sociales desde las 7am"

**Backend debe:**
- Parsear hora futura ("hoy a las 3", "mañana a las 9am")
- Crear recordatorio programado
- A la hora indicada: enviar recordatorio + activar bloqueos
- Tasker ejecuta bloqueo y reproduce mensaje

### 3. Turnos Rotativos (Solo Bajo Demanda)
**NO se bloquean automáticamente**

**Comandos de voz:**
- "ARGOS activa el turno 1" → Bloquea apps según configuración del turno
- "ARGOS desactiva bloqueos" → Quita todos los bloqueos activos
- "ARGOS crea un turno nuevo de 8am a 4pm con bloqueo de YouTube e Instagram"

**Backend debe:**
- Gestionar múltiples turnos configurables
- Activar/desactivar turnos por comando
- Permitir crear nuevos turnos dinámicamente

### 4. Bloqueo de Sitios (Comandos Simples)
**Comandos de voz:**
- "ARGOS bloquea páginas" → Sistema entiende que son páginas porno
- "ARGOS bloquea redes sociales"
- "ARGOS desbloquea todo"

**Backend debe:**
- Mantener listas predefinidas (pornografía, redes sociales, etc.)
- Activar/desactivar bloqueos por categoría
- Agregar sitios custom: "ARGOS bloquea reddit.com"

## Tecnologías Requeridas

### NLP y Procesamiento de Comandos
- **LLM API** (Abacus AI / OpenAI): Parsear comandos de voz complejos
- Extraer: acción, hora, fecha, duración, apps/sitios a bloquear
- Convertir lenguaje natural a structured data

### Sistema de Tareas Programadas
- **Cron Jobs externos** (Abacus AI cron system): Para alarmas futuras
- DB almacena: alarmas pendientes con timestamp
- Cron job verifica cada 5 minutos si hay alarmas que ejecutar

### Base de Datos
- **Turnos**: id, nombre, hora_inicio, hora_fin, apps_bloquear[], sitios_bloquear[]
- **Alarmas**: id, usuario, timestamp, mensaje, tipo (recordatorio/bloqueo/turno)
- **Bloqueos_Activos**: id, apps[], sitios[], timestamp_inicio, duracion, trigger
- **Comandos_Voz**: id, texto_comando, parsed_data, timestamp (para historial)

## Arquitectura Actualizada

### Flujo 1: Comando de Voz con Hora Futura
```
Usuario: "ARGOS hoy a las 3 voy a estudiar, recuérdame y bloquea todo"
↓
Tasker captura audio → envía a backend
↓
Backend usa LLM para parsear:
  - accion: "recordatorio + bloqueo"
  - fecha: "hoy"
  - hora: "15:00"
  - apps_bloquear: ["instagram", "tiktok", "youtube"]
↓
Backend crea alarma en DB con timestamp: hoy 15:00
↓
Backend responde a Tasker: "Alarma creada para las 3pm"
↓
Tasker reproduce: "Alarma creada para las 3 de la tarde"
↓
[Cuando llega las 3pm]
Cron job detecta alarma pendiente
↓
Backend envía webhook a Tasker con comando de bloqueo
↓
Tasker bloquea apps y dice: "Es hora de estudiar, apps bloqueadas"
```

### Flujo 2: Comando Simple de Bloqueo Inmediato
```
Usuario: "ARGOS bloquea páginas"
↓
Backend entiende: categoría = "pornografía"
↓
Backend consulta lista predefinida de sitios porno
↓
Backend crea bloqueo activo con sitios[]
↓
Backend envía comando a Tasker
↓
Tasker bloquea sitios y confirma por voz
```

### Flujo 3: Agregar Alarma para Turno
```
Usuario: "ARGOS agregar alarma a las 4:15am para turno 1"
↓
Backend parsea:
  - tipo: "alarma_turno"
  - hora: "04:15"
  - turno_id: 1
↓
Backend crea alarma en DB
↓
Backend crea cron job que se ejecuta a las 4:15am
↓
A las 4:15am: Cron job activa turno 1
↓
Tasker recibe comando y activa bloqueos del turno
↓
Tasker reproduce: "Turno 1 activado"
```

## Endpoints API Necesarios

### Comandos de Voz
- POST /api/comando-voz
  - Body: { audio_base64 } o { texto }
  - LLM parsea comando
  - Ejecuta acción inmediata o programa alarma
  - Response: mensaje de confirmación

### Alarmas
- POST /api/alarmas
  - Body: { fecha, hora, tipo, mensaje, metadata }
  - Crea alarma programada + cron job
- GET /api/alarmas/pendientes
  - Lista alarmas futuras del usuario
- DELETE /api/alarmas/:id
  - Cancela alarma

### Turnos
- POST /api/turnos
  - Crea turno nuevo con horario y bloqueos
- POST /api/turnos/:id/activar
  - Activa turno manualmente
- POST /api/turnos/:id/desactivar
  - Desactiva turno

### Bloqueos
- POST /api/bloqueos/categoria
  - Body: { categoria: "pornografia" | "redes_sociales" }
  - Activa bloqueo de categoría predefinida
- DELETE /api/bloqueos/todos
  - Desactiva todos los bloqueos

### 🔧 MÓDULO DEVICE DIAGNOSTICS & OPTIMIZATION
**4 Endpoints:**
1. `POST /api/v1/device-diagnostics/register` - Registrar diagnóstico
2. `GET /api/v1/device-diagnostics/latest` - Último diagnóstico
3. `GET /api/v1/device-diagnostics/history` - Historial
4. `POST /api/v1/device-diagnostics/clear-cache` - Limpiar caché

**Métricas Monitoreadas:**
- RAM total y disponible
- CPU (porcentaje)
- Temperatura del dispositivo
- Batería (%)
- Almacenamiento libre

**Problemas Detectados Automáticamente:**
- ⚠️ RAM baja (< 500MB)
- ⚠️ CPU alta (> 80%)
- 🔥 Temperatura alta (> 40°C)
- 🔋 Batería baja (< 20%)
- 💾 Almacenamiento casi lleno

**Recomendaciones Inteligentes:**
- "Cierra apps en background para liberar RAM"
- "El dispositivo está caliente, déjalo enfriar"
- "Activa modo ahorro de batería"
- "Elimina fotos/videos antiguos"
- "Reinstala la app X que consume mucho CPU"
